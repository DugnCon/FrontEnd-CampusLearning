CampusLearning – Documentation Hub

Platform Overview

CampusLearning is a modular learning-management ecosystem composed of several Single-Page Applications (SPAs) and micro-services. The table below gives you a quick taste of what each piece does.

Frontend Applications

  • User App

    Student interface with AI Chat, Monaco editor, competitions and real-time notifications.

  • Student Web App

    Campus Learning-specific portal: dashboard, course registration, grade tracking & timetable.

  • Teacher App

    Manage courses, publish lessons, build assignments, live gradebook & analytics.

  • Admin App

    System configuration, user/role management, site-wide monitoring.

  • Student Admin App

    Deep student record management, academic catalogue & finance dashboards with CSV import/export.

Backend Services

  • User Service

    Auth, profiles, courses, competitions & websocket notifications.

  • Student Service

    Academic data, schedules, tuition & exam-registration APIs (with demo fallback).

  • Teacher Service

    Course CRUD, assignments, grading endpoints. JWT-protected.

  • Admin Service

    System-wide admin APIs: users, courses, events, reports, settings.

  • Student Admin Service

    Student CRUD, academic catalogue, finance, service desk.

  • Judge0 Master

    Containerised code runner for 50+ languages.

  • Code-Server

    Browser-based VS Code for interactive labs.

Core Platform Capabilities

  • Real-Time Collaboration

    Socket.IO powers live chats, notifications and coding sessions.

  • AI Assistance

    Gemini API integration provides contextual coding help & study advice.

  • Secure Auth

    JWT + refresh tokens, role-based access and granular middleware.

  • Containerised Execution

    Judge0 + Docker ensure safe, consistent multi-language code runs.

  • Scalable Architecture

    Micro-services with independent deploy pipelines & health checks.
